# Git — everyone

The single home for branching, commits, and history — variants point here instead of restating it.

## Branches
- **`main` is protected** everywhere — no direct pushes, no force-push, ever. Land changes through a branch + PR.
- **Branch off `main`,** keep it short-lived, name it `<type>/<slug>` by **what the change adds, not the files touched**:

| Type | For | Merged? |
|---|---|---|
| `feat` | new capability | yes |
| `bug` | fix wrong/regressed behaviour | yes |
| `refactor` | restructure, no behaviour change | yes |
| `chore` | housekeeping — deps w/o API change, formatting, config | yes |
| `spike` | investigate *"is this external tool fit?"* — throwaway | no |
| `poc` | prove our own design assembles & runs — throwaway | no |
| `hotfix` | emergency fix to `main` (maintainer/owner only) | yes |

`spike` = is the tool right (external); `poc` = does our design hold (internal). Spike code never becomes poc code.

## Commits
- Small, focused; imperative subject prefixed with the type — `feat(scope): add X`. Say *why* in the body when non-obvious; single quotes for heredoc ergonomics.
- Never commit secrets, tokens, or credentials.
- No AI-marker lines in history (`Co-authored-by`, "🤖", "Generated by …") — keeps commits portable; `strict` enforces it hardest.

## Flow by variant
- **solo** — branch → PR → self-merge into `main` once CI is green. No second reviewer, no `dev`/`release` chain; the type taxonomy is optional.
- **team** — promotion chain over protected `main`, `release/*`, `dev`; every PR to a protected branch needs green lint + test + review:
  ```
  feat|bug|refactor|chore/*  →  dev  →  release/*  →  main
  spike|poc/*                →  not merged
  hotfix/*                   →  main   (maintainer/owner only)
  ```
- **strict** — team flow, plus: don't commit AI tool config (`CLAUDE.md`, `.cursor/`, `.aider*`, copilot instructions); load rules from `~/.claude/` so the repo stays NDA-portable.
